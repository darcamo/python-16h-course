Unit Tests

  • Unit tests são um tipo particular de testes onde a menor funcionalidade do seu programa é testada
  • Cada unidade de teste deve ser completamente independente
  • Cada teste deve poder rodar de maneira isolada ou como parte de uma "suíte" de testes
    • A ordem de execução dos testes não deve importar
    • Cada teste deve importar seu próprio conjunto de dados independentes dos demais testes
  • Os testes devem rodar rapidamente e de maneira automatizada
  • Ao identificar um bug no seu programa, crie um teste que reproduza o bug só depois conserte o bug

Unit Tests

O módulo unittests do python

  • Python possui o módulo unittests
  • Para criar novos casos de teste criamos uma subclasse de unittest.TestCase
  • Estrutura básica para testes
            import unittest

def fun(x):
    return x + 1

class MyTest(unittest.TestCase):
    def test_something(self):
        self.assertEqual(fun(3), 4)
